# Robustness to Text Transformations

This README describes the code used for the robustness analysis in the appendix section **Robustness to text transformations** of our paper.

This analysis is based on the official implementation of:

**MARKMYWORDS: Analyzing and Evaluating Language Model Watermarks**

Official repository: https://github.com/wagner-group/MarkMyWords

For the detailed definitions and implementation details of the original text transformation attacks, please refer to the official `MarkMyWords` repository and the original paper. In this directory, the original README from the official repository has been renamed as `README_old.md`.

We modify the original repository only to adapt the benchmark attacks to the data structure used in our experiments. Unless explicitly stated below, we do not change the original functionality of the attack methods.

## 1. Basis of Modification

We start from the official `MarkMyWords` repository and modify files under:

`/src/watermark_benchmark/attacks/`

Specifically, we adapt the following original files:

`helm_attacks.py`

`paraphrase_attack.py`

`swap_attack.py`

`synonym_attack.py`

The modified versions are saved with the suffix `_new` for distinction:

`helm_attacks_new.py`

`paraphrase_attack_new.py`

`swap_attack_new.py`

`synonym_attack_new.py`

These `_new` files are adapted to support the data format used in our test-time evaluation. Apart from this compatibility adaptation, the attack functionalities are kept unchanged.

## 2. Added Scripts

In addition to the modified attack files, we add several scripts under:

`/src/`

The added files are:

`perturb_watermarked_homo.py`

`perturb_watermarked_jsonl.py`

`homo_perturbed_char_summary.py`

`wm_perturbed_summary.py`

`run_attack.sh`

`alphabet.txt`

Their functions are described below.

### `perturb_watermarked_homo.py`

This script applies text transformations to the baseline dataset.

It uses the adapted attack functions to perturb watermarked text generated by the baseline method and saves the transformed outputs for later robustness analysis.

### `perturb_watermarked_jsonl.py`

This script applies text transformations to our own dataset.

It processes the watermarked data used by our method, performs the same set of text transformations, and saves the perturbed outputs.

### `homo_perturbed_char_summary.py`

This script analyzes the perturbation results of the baseline dataset.

It summarizes how the text transformations affect the baseline watermark signal after perturbation.

### `wm_perturbed_summary.py`

This script analyzes the perturbation results of our method.

In particular, it checks the preservation and removal of invisible Unicode characters after text transformations. The file `alphabet.txt` is used by this script to identify and analyze the invisible characters used in our watermark.

### `run_attack.sh`

This shell script provides a convenient way to run the full perturbation and analysis pipeline.

It runs the following four Python scripts:

`perturb_watermarked_homo.py`

`perturb_watermarked_jsonl.py`

`homo_perturbed_char_summary.py`

`wm_perturbed_summary.py`

### `alphabet.txt`

This file contains the invisible character alphabet used by our watermarking method.

It supports `wm_perturbed_summary.py` in counting and analyzing the preservation of invisible characters after text transformations.

## 3. Output Directory

All text transformation results and corresponding analysis results are saved under:

`/output/`

This directory contains both the perturbed text files and the summarized robustness analysis results.

## 4. Purpose of This Analysis

The purpose of this code is to evaluate whether watermark signals remain detectable after common text transformations.

In the appendix of our paper, this analysis is used to support the section:

**Robustness to text transformations**

The experiments compare the behavior of baseline watermarking methods and our invisible Unicode-based watermarking method under different transformations, such as paraphrasing, word swapping, synonym replacement, and other perturbations adapted from the `MarkMyWords` benchmark.

Specific details of the text transformations are not repeated in this README. Please refer to the original `MarkMyWords` paper and the original repository documentation, which is preserved in this directory as `README_old.md`.

## 5. Summary of Changes

Compared with the official `MarkMyWords` repository, the main changes are:

1. We adapt four attack files under `/src/watermark_benchmark/attacks/` to support our test-time data structure.

2. The adapted files are saved with the `_new` suffix.

3. We add two perturbation scripts for applying text transformations to the baseline dataset and our own dataset.

4. We add two summary scripts for analyzing the perturbation results.

5. We add `run_attack.sh` to run the complete pipeline.

6. We add `alphabet.txt` to support invisible-character analysis for our method.

7. All perturbation outputs and analysis results are saved under `/output/`.

No additional changes are made to the core functionality of the original attack methods.
